The first thing I did was to use the psycopg2 Python module to connect to the PostgreSQL database, define a cursor and execute a SELECT statement. The most important line of this process is the following:

cur.execute( "SELECT * FROM public.bvb_data WHERE UPPER(symbol) = UPPER(%s)", (symbol,) )

This line, uses a cursor, named "cur" in order to execute the SELECT statement contained by paratheses.

After I managed to perform that SELECT, I fetched all that data and saved it inside a list of tuples using: rows = cur.fetchall()

With this list, it was extremely easy to display that data. I simply sent this list, as input parameter, to the rendered HTML file and displayed that data inside a table. So far, I learned about how to check the kernel activity and different info (like the kernel version), using dmesg, /proc or uname utilities. Another interesting thing that I learned here is the 'tainted kernel'. A 'tainted kernel' is a kernel that works with closed source drivers. Closed source drivers are drivers that do not have the source code made public by the hardware vendor.

There are 2 types of drivers: open source and closed source drivers. The open source drivers have the code published on the Internet, by the vendor of the hardware you are using. If problems related to the open source drivers appear, the community can inspect the source code of the drivers and come up with a bug fix which will be solved during the next patch.

The closed source drivers do not have the source code published. The drivers are distributed in a binary format. If problems occur, coming up with a solution will be an extremely hard thing as you do not have access to the source code. This is why I think, Linus Torvalds "appreciates" NVIDIA XD (link: https://www.youtube.com/watch?v=iYWzMvlj2RQ), because they do not publish the drivers source code for their chips. They distribute it directly compiled.

Even though it is hard so solve this kind of problems, a solution exists. The 'tainted kernel' thing I mentioned above is the solution. In case of a tainted kernel (a kernel which uses closed source drivers), if a driver causes problems, the kernel will suggest you to remove it in order to solve that problem. Pretty easy, right? =)))))

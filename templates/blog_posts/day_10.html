The technical achievement


Moving on to the real deal now. I really need to tell you about this achievement. Everything started with me learning about the storage management of the Linux RedHat 9 OS. This subject had 2 chapters where this topic was discussed. The first chapter (Chapter 14 - Managing Storage) focused on creating partitions (MBR and GPT) using fdisk, gdisk and parted. It was interesting to see how they can be created and configured. The other chapter (Chapter 15 - Managing Advanced Storage) focused on LVMs and how you can create and configure them. This chapter was way more interesting to me because I had the chance to learn about how can I create a Physical Volume using disks or partitions (which I learned how to create and configure in chapter 14). Another thing that I learned here was to create a Volume Group out of those Physical Volumes and then create Logical Volumes (LVMs) using the Volume Group. What is interesting about Logical Volumes is that their size is flexible. When a logical volume fills up, you can easily add more space from the volume group it is part of. No enough space on the volume group to extend the logical volume? Add a new physical volume. No more physical volumes? Add more disks/partitions. This is the thing that I like the most about this storage management approach. Another thing that I like here is that you have the chance to enforce some additional security good practices in order to secure partitions using the /etc/fstab file.

For now, the application is simplistic. I just connect to the bvb.ro and collect some statistical data of some stocks I like, using Selenium. After that, I save that data in a python dictionary, I create the insert statement and I populate a table inside the PostgreSQL database with the data collected, using the psycopg2 module. This is the way I store the data inside my PostgreSQL. You may ask now: "But, Andrei, how is all of this related with the storage management part?". Well, when I configured the PostgreSQL database, I made sure that each of the WAL files, the logs and the data folder of the PostgreSQL database are stored inside separate LVMs which are mounted on specific locations:

  • The pg_wal is stored inside the /mnt/pg_wal which is the folder where I mounted and ext4 file system LVM. This lvm will contain only the WAL files.
  • The pg_data is stored inside the /mnt/pg_data which is the folder where I mounted the ext4 file system LVM. Here I save the data folder of the PostgreSQL database.
  • The logs of the PostgreSQL database are stored inside the /mnt/pg_logs folder. This is where I mounted another ext4 file system. In this LVM I keep all the logs related to the PostgreSQL database.

I know that these components of the PostgreSQL database should be kept on different storage environments, as a good practice, and this is basically what I tried to do here.

At the moment, I set up a cron job that uses this project to populate the database every 2 minutes so that I can observe how the usage of the lvms increases.

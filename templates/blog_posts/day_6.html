I think that today I will try to create some Ansible tasks from the chapter 3 and document them in the README.md file of the task. One great thing that I learned these days is that when creating a new role inside your project you can use the following command: ansible-galaxy role init your_role_name for that. This is extremely useful because it generates you a role directory structure out of the box, a structure that contains a README.md file and var, handles and tasks directories, with some .yml file inside. This is a pretty neat feature of the Ansible that I am and certainly use in the future.

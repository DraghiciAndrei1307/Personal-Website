Introduction


Hello! This day (January 25, 2026) finds me again in a coffee shop (a Starbucks coffee shop this time) trying to learn something that is quite important for me at me moment (I 'll get to this later). This last week, my direct manager decided that it would be a good idea to get me involved in a project that aims to create a Django API that stands as a layer between the applications the firm uses and the databases stored in our Cloud.

And so, this Friday (January 23, 2026), I met Michael, who is the guy that coordinates this whole solution. He is a nice guy and is a senior software developer from Germany. He said that he and another colleague are working at this solution, and they are in need for some additional help. He also said that they are using Django Rest Framework to develop the API and granted us access to their repository and documentation. Overall, it was a good meeting, with a lot of insights presented, a meet that stirred my interest.

To be honest, I am not a software developer (in the true sense of the word). I am merely a rookie in software development . I mean, I did work at some projects/homeworks related to the uni which involved a bit of Git and GitHub, but never worked in a true software development environment, with specialists like Michael. I know how to init a Git project, to check status (using the git status command), to add the changes, commit them and push them to the GitHub repo. I also know how to work with branches and merge (using a pull request) the work I perform there to the main branch. I also know how to generate SSH keys and add them to GitHub in order to interact with the repo in a more secure way. But, even though I know all of this, I still feel like I have a lot to learn about Git and GitHub in order to feel ready to work in a proffesional environment. I really want to make a good impresion to Michael and prove (to him and myself) that I can work in a clean and structured manner.

And so, I sketched a plan to prepare myself for working at the API solution that Michael is in charge of. The plan is as follows:

  • Become proficient in Git and GitHub as fast as possible. For this, I started following a Git and GitHub course from Udemy which aims to get you through all the relevant topics of Git and GitHub, and more (Link: https://www.udemy.com/course/git-and-github-bootcamp/).
  • Follow the Quickstart section of the Django documentation.
  • Create a project that uses the things learned from the Django documentation. I am thinking of two VMs. On one VM, I will run the application which will use the Django Rest API to manipulate the PostgreSQL database from the other VM. For the start, I will perform a few SQL statements (via API calls). I guess it is enough for a starter.

I think that my plan for the next few weeks will be a good proof of concept, if I wrap this up in a clean, structured and well-documented GitHub repository.

Until now, I managed to get through 8 sections out of 20. I think I managed to learn and understand the core concepts of Git. The videos are pretty easy to follow and I like that almost each section has some exercises in order to practice the topics discussed. So far, I am pretty satisfied with this course. I learned a lot of things that before I thought I mastered. This course helped me to structure the knowledge I thought I had. I am glad that finally I had the chance to learn this subject the right way. For example, I learned how to solve conflicts between two branches (a thing that before resembled with Chinese to me), how to perform a merge commit (if there were no conflicts) and how to use git diff in order to check the changes between 2 commits, 2 branches and also check what changes were made (staged, unstaged or both) since the last commit of a specific branch.

Now, I am trying to complete in this course, to learn more and more, but I feel just a little bit tired. I guess this is some cognitive fatigue that is the outcome of the last days or weeks of learning and developing daily. I mean, at the moment I have a lot of projects ongoing. These projects are:

  • Reading about the Linux RedHat 9 Operating System - the aim here is to learn to use and manage it in order to create an environment to run my projects
  • RedHat Automation with Ansible - the aim here is to use Ansible to automate different tasks performed on a Linux RedHat environment (with this project, I shoot two birds with one stone: I recap the RedHat knowledge and I practice Ansible)
  • Trading App - this app aims to help me determine if a stock is worth investing my money into. For a long time I desired to create such app. I am reading this book "Intelligent Investor" which helps me understand how to become an intelligent investor, how secure my money and beat the inflation at the same time (which is also a way to secure your assets).
  • Home-related tasks. Cooking, cleaning and fixing stuff takes a lot of time, but I cannot work in an environment that is not clean. I like to have everything in place. Otherwise, I cannot fully concentrate.
  • Work - this is a challenging topic. I have a lot of work to do there. A lot of databases to be managed, systems to be scaled up/debugged and documented.

Now that I have showcased news and plans related to my life, I feel prepared to resume my learning process of understanding Git and GitHub. I really feel like creating posts (basically journaling my thoughts) helps me to feel productive. It makes me eager to work on my projects. It simply gives me that boost of motivation that sometimes is missing.

I wish you all the best and I hope you have a great day and even greater achievements. Cheers!

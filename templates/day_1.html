Coming back to the subject of today's discussion

Today I learned about how to schedule jobs on a server provisioned with RedHat 9. There are 3 ways you can achieve that:

  • Using Systemd timer units
  • Using the old, but awesome, crond
  • Using the at utility

I am going to say a few words about each one of them in the following section.

The Systemd timers

Here are some interesting ideas:

  • The Systemd timers are the most recommended way to schedule jobs in RedHat 9 distribution.
  • A Systemd timer is always used alongside a service unit file. The timer decides WHEN the service should be started and the service unit decides HOW to start the service.
  • If you need a service to be started by a timer, you enable the timer, NOT the service.

Scheduling jobs using cron

Here are some interesting ideas:

  • You can still schedule jobs in RedHat 9, using cron.
  • The crond daemon is started as a systemd service. The crond looks every minute to see if it is work to do. I mean, it checks the cron configuration in order to start the right service at the right time.
  • It does not need to be restarted or reloaded. The crond daemon wakes up every minute and checks its config to see if there is something to do.

Scheduling jobs using at

Here are some interesting ideas:

  • Even if you are using cron to schedule jobs, you can also use the at, to be more precise, the atd daemon to schedule jobs that will be executed only once.
  • To schedule a job with this utility, just type at command followed by the time when it must be executed. One example can be: at 15:00. After you pressed enter, the at shell will open and you can type down your commands. In the end, press Ctrl+D to quit.
  • To see all the jobs scheduled with atd, you can type atq. To remove jobs, use the atrm, followed by the number of the job you want to remove.
